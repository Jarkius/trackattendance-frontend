### Key Code Changes
- `config.py`: Replaced `sys.exit(1)` with warning message when no API key → local-only mode
- `main.py:admin_set_api_key()`: Validates key, stores in SQLite, initializes SyncService at runtime
- `main.py:load_saved_settings()`: Loads API key from SQLite if .env is empty
- `main.py`: Conditional SyncService/DashboardService initialization (only when API key present)
- `attendance.py:search_employee()`: 3-tier search: exact substring → all-words-any-order → fuzzy (SequenceMatcher ≥ 0.75)
- `attendance.py:_fuzzy_word_score()`: Word-level similarity scoring using difflib.SequenceMatcher
- `attendance.py:_safe_string()`: Changed from `.strip()` to `" ".join(s.split())` to collapse internal whitespace
- `PLAN-commercialize.md`: Added 13 competitors, 12 pain points, buyer personas, regional pricing, per-event packs, SEO keywords, revenue projections

### Architecture Decisions
- **3-tier search over pure fuzzy**: Exact matches are always preferred. Fuzzy only activates when no exact/word matches found — prevents polluting results
- **SQLite API key storage over .env rewriting**: Safer (no file permission issues), survives .env replacement, follows existing settings pattern
- **No restart required for API key**: SyncService initializes on-demand when key is first entered, using existing db and config
- **Per-station licensing over device-locking**: No hardware fingerprint = zero support burden when kiosks replaced

## Lessons Learned
- **Pattern**: 3-tier search (exact → word → fuzzy) is the right architecture for name lookup — each tier has clear semantics and fallback behavior
- **Discovery**: Python's `difflib.SequenceMatcher` is sufficient for typo-tolerant name matching at small scale (< 10K employees) without external dependencies
- **Pattern**: `" ".join(s.split())` is the idiomatic Python way to normalize all whitespace in one operation
- **Discovery**: The attendance tracking market has a clear "missing middle" between free tools and $7,800+/year enterprise — $20-100/month is underserved
- **Pattern**: Conditional service initialization (only when API key present) enables graceful degradation without code path explosion
