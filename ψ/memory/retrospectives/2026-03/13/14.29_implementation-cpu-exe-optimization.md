### Key Code Changes

- Kept voice/greeting assets bundled in spec (already have exe-adjacent override support)
- onedir over onefile: faster startup is worth the folder deployment tradeoff

### Friction Points
1. **CWD reset**: Every bash command resets to archive directory. Manageable but adds noise.
2. **Test helper coupling**: `ProximityDetector.__new__` bypasses `__init__`, so new attributes must be manually added to the test helper. Fragile pattern.
3. **No build verification**: Can't actually run PyInstaller on macOS to verify Windows exe build. Must trust the spec syntax.

## Lessons Learned
- **Pattern**: Planning-heavy sessions enable implementation-light follow-ups. 4 hours of analysis → 11 minutes of correct code.
- **Discovery**: Test helpers that bypass `__init__` via `__new__` are fragile — they break whenever a new instance attribute is added. Consider using the constructor with mocked dependencies instead.
