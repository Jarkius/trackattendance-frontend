### Architecture Decisions
- Force-set (`os.environ[key] = value`) over `setdefault` for test env vars when `.env` file exists — `load_dotenv()` populates env before test setup runs
- Extracted BU computation as standalone function for testing rather than testing through full `export_to_excel()` — avoids openpyxl/database/API dependencies in unit test

## Lessons Learned
- **Pattern**: Use `os.environ[key] = value` (force-set) in test setup when `load_dotenv()` may have loaded values from `.env` — `setdefault` only works if the key doesn't exist yet
- **Discovery**: Extracting embedded business logic into standalone functions for unit testing is the right pattern when the host function has heavy I/O dependencies
- **Pattern**: Cloud validation after all code merges catches integration issues that unit tests can't — running export against 390 real scans confirmed BU counts match
